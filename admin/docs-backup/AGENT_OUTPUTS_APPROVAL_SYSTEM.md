# Agent Outputs & Approval System

## Overview

The Agent Outputs & Approval System is a comprehensive dashboard that allows you to review, approve, or reject all content and strategies generated by your AI agents. It includes a learning system that helps agents improve based on your feedback.

## Features

### 🎯 **Daily Output Review**
- View all agent outputs for any date
- Filter by agent, status, or output type
- Real-time statistics and metrics
- Priority-based sorting

### ✅ **Approval/Rejection System**
- Approve outputs with optional feedback
- Reject outputs with required reasons
- Track approval history and patterns
- Bulk operations for efficiency

### 🧠 **Learning & Improvement**
- Agents learn from your feedback
- Automatic improvement suggestions
- Performance tracking and trends
- Pattern recognition for common issues

### 📊 **Analytics & Insights**
- Agent performance metrics
- Approval rate tracking
- Common rejection reasons
- Improvement trends over time

## How to Use

### 1. Access the Dashboard

Visit: `http://localhost:3000/dashboard/agent-outputs.html`

Or navigate from the main dashboard using the "Agent Outputs" link.

### 2. Review Daily Outputs

The dashboard automatically shows today's outputs. You can:

- **Filter by Date**: Select any date to review historical outputs
- **Filter by Agent**: Focus on specific agents (CMO Brain, Data Analyst, etc.)
- **Filter by Status**: View pending, approved, or rejected outputs
- **Filter by Type**: Sort by campaign, content, strategy, analysis, or report

### 3. Review Individual Outputs

For each output, you can:

- **View Full Content**: Click "View Full" to see the complete output
- **Approve**: Click "Approve" to accept the output
- **Reject**: Click "Reject" to reject with a reason
- **Provide Feedback**: Add optional feedback to help agents learn

### 4. Approval Process

#### Approving an Output:
1. Click "Approve" on any pending output
2. Optionally add feedback (recommended for learning)
3. Click "Approve" to confirm
4. The output status changes to "Approved"

#### Rejecting an Output:
1. Click "Reject" on any pending output
2. **Required**: Provide a reason for rejection
3. Optionally add additional feedback
4. Click "Reject" to confirm
5. The output status changes to "Rejected"

### 5. Learning System

The system automatically:

- **Records Feedback**: All approvals and rejections are stored
- **Analyzes Patterns**: Identifies common rejection reasons
- **Generates Improvements**: Suggests specific improvements for each agent
- **Tracks Performance**: Monitors approval rates and trends

## Learning System Details

### How Agents Learn

1. **Feedback Collection**: Every approval/rejection is recorded with context
2. **Pattern Analysis**: System identifies common rejection reasons
3. **Improvement Generation**: Specific suggestions are created for each agent
4. **Context Integration**: Learning is applied to future outputs

### Learning Context Includes

- **Recent Feedback**: Last 5 approvals/rejections with reasons
- **Improvement Suggestions**: Specific areas to focus on
- **Agent-Specific Instructions**: Tailored guidance for each agent type
- **Performance Trends**: Approval rates and improvement patterns

### Example Learning Context

```
LEARNING CONTEXT FOR COPYWRITING AGENT:

RECENT FEEDBACK:
❌ REJECTED: Content too generic, needs more specific local business focus
   Feedback: Add specific examples for local businesses
✅ APPROVED: Great tone and length, clear call to action

IMPROVEMENT SUGGESTIONS:
• Make content more specific to local business context
• Include specific examples relevant to target industries
• Focus on local business pain points and solutions

AGENT-SPECIFIC INSTRUCTIONS:
• Focus on local business pain points and solutions
• Use conversational, benefit-focused language
• Include specific examples relevant to target industries
```

## API Endpoints

### Get Outputs
```
GET /api/approval/outputs?date=2025-07-28&agent=data-analyst&status=pending&type=analysis
```

### Approve Output
```
POST /api/approval/{outputId}/approve
{
  "approvedBy": "Hillary McMullen",
  "feedback": "Great analysis, very actionable",
  "outputType": "analysis"
}
```

### Reject Output
```
POST /api/approval/{outputId}/reject
{
  "rejectedBy": "Hillary McMullen",
  "reason": "Content too generic",
  "feedback": "Need more specific local business examples",
  "outputType": "content"
}
```

### Get Learning Data
```
GET /api/approval/learning/copywriting-agent
```

## Troubleshooting

### No Outputs Found
- Run `node discover-agent-outputs.js` to import existing files
- Check that agent output files exist in your project directory
- Verify file naming patterns match expected formats

### Dashboard Not Loading
- Ensure server is running on port 3000
- Check browser console for errors
- Verify API endpoints are accessible

### Learning Not Working
- Check that feedback is being saved
- Verify agent names match expected values
- Review learning system logs
